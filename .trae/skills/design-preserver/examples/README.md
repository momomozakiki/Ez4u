# Design Preservation Examples

This directory contains purpose-specific examples for design preservation patterns.

## Available Examples

- **[styling-patterns.md](styling-patterns.md)** - How to detect and preserve Tailwind vs custom CSS approaches
- **[component-patterns.md](component-patterns.md)** - Props API, composition, and naming conventions
- **[destructive-changes.md](destructive-changes.md)** - When changes must break patterns and approval requirements
- **[verification-output.md](verification-output.md)** - Copy-paste templates for chat output

## Usage

Each example file focuses on a specific preservation scenario:
1. **Detection** - How to identify existing patterns
2. **Preservation** - How to extend without breaking
3. **Validation** - How to verify changes are safe
4. **Output** - What messages to show users
